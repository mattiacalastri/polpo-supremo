## Perche vibe-guard primo, non open-memory

- open-memory richiede un cambio mentale (file-based memory e controcorrente vs vector DBs)
- vibe-guard e un pain acuto e immediato: "vibe coding" e trending, security su AI code e vuoto
- un security scanner si viral su HN e Twitter dev in modo naturale
- apre la porta a open-memory come "quello che viene dal team di vibe-guard"

## GitHub Org
